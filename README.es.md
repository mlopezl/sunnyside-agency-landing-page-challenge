# Frontend Mentor – Solución de la Landing Page de Sunnyside Agency

Esta es mi solución al reto **Sunnyside Agency Landing Page** de Frontend Mentor. El proyecto se centra en la construcción de una landing page moderna y responsive para una agencia creativa, replicando fielmente el diseño proporcionado y siguiendo buenas prácticas de frontend limpias, mantenibles y escalables.

Este reto fue una excelente oportunidad para practicar layouts responsivos, CSS Grid y Flexbox, manejo de estados de la interfaz y uso de JavaScript vanilla para la interactividad.

## Descripción general
Este proyecto es una landing page responsive para una agencia creativa que se adapta perfectamente a dispositivos de escritorio, tablet y móvil. Incluye una sección hero a ancho completo, secciones de servicios, testimonios, una galería de imágenes y un footer con enlaces sociales.

La página cuenta con un menú de navegación optimizado para móviles controlado mediante un icono hamburguesa, cambios de estado visuales suaves y uso de imágenes responsivas adaptadas a distintos tamaños de pantalla.

El objetivo fue replicar el diseño original de Frontend Mentor manteniendo un código semántico, ordenado y fácil de mantener.

---

## El reto
Los usuarios deben poder:

- Ver el layout óptimo según el tamaño de pantalla de su dispositivo.
- Disfrutar de un diseño totalmente responsive en escritorio, tablet y móvil.
- Abrir y cerrar el menú de navegación móvil usando un icono hamburguesa.
- Cerrar el menú móvil haciendo clic fuera de él.
- Ver estados hover en elementos interactivos como botones, enlaces e iconos sociales.
- Navegar una interfaz que se asemeje lo más posible al diseño original de Sunnyside.

## Mi proceso
- Estructuré la página utilizando **HTML5 semántico** con elementos como `header`, `nav`, `main`, `section`, `article` y `footer`.
- Construí el layout principal usando **CSS Grid** para las secciones grandes y **Flexbox** para la navegación y alineación de componentes.
- Implementé **diseño responsive** mediante múltiples breakpoints para garantizar una buena experiencia en todos los dispositivos.
- Utilicé **propiedades personalizadas de CSS (variables)** para gestionar los colores y mantener consistencia visual.
- Seguí la **metodología BEM** para el nombrado de clases, facilitando la escalabilidad y el mantenimiento del CSS.
- Implementé un menú de navegación responsive con **JavaScript vanilla**, controlando los estados de la interfaz mediante clases CSS.
- Apliqué el patrón de UX de “clic fuera para cerrar” el menú móvil.
- Gestioné imágenes separadas para desktop y mobile con el fin de optimizar el layout y la fidelidad visual.
- Añadí estados hover y activos para mejorar la usabilidad y la interacción del usuario.

## Lo que aprendí
- Cómo construir una landing page limpia y responsive siguiendo una especificación de diseño.
- Combinar de forma efectiva **CSS Grid y Flexbox** dentro de un mismo proyecto.
- Gestionar estados de la interfaz (abrir/cerrar menú móvil) usando JavaScript vanilla y clases CSS.
- Implementar patrones comunes de UX como cerrar un menú al hacer clic fuera de él.
- Mejorar la mantenibilidad del código mediante **variables CSS** y **convenciones BEM**.
- Manejar imágenes responsivas para distintos tamaños de pantalla.
- Estructurar proyectos frontend de forma escalable y cercana a entornos reales de producción.
